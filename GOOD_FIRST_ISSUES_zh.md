# 适合新贡献者的 Issue

这些入门任务直接来源于 OpenClaw 改进规范，目标是帮助新贡献者以清晰、可控的范围做出有价值的推进。

## 1. 为注册表条目添加规范化 ID

- **对应规范轨道：** Track A
- **目标：** 为命令和工具条目引入稳定、带命名空间的 ID
- **意义：** 减少歧义，让路由和审计更安全
- **建议产出：** 数据模型更新，以及一条重复 ID 校验路径

## 2. 增加路由可解释性输出

- **对应规范轨道：** Track B
- **目标：** 为路由决策增加适合调试的解释格式
- **意义：** 让错误路由更容易被看见和诊断
- **建议产出：** 包含分数或原因拆解的候选列表

## 3. 生成成熟度级别报告工件

- **对应规范轨道：** Track C
- **目标：** 为工具或功能成熟度生成机器可读报告
- **意义：** 把模糊的对等性描述转化为可衡量状态
- **建议产出：** 输出 L0-L4 状态的 CLI 结果或工件文件

## 4. 在详细模式中暴露策略原因代码

- **对应规范轨道：** Track D
- **目标：** 在诊断信息中输出结构化的允许/拒绝原因代码
- **意义：** 提升可支持性、信任度和治理清晰度
- **建议产出：** 显示策略原因的结构化 verbose 事件或调试文本

## 5. 搭建模式契约测试脚手架

- **对应规范轨道：** Track E
- **目标：** 为 remote、SSH、teleport 或 direct 模式建立第一版测试
- **意义：** 防止不同运行模式之间的行为漂移
- **建议产出：** 测试骨架和一份失败类型说明

## 建议标签

- `good first issue`
- `help wanted`
- `documentation`
- `enhancement`
- `governance`
