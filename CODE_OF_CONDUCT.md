# Code of Conduct

## Our Commitment

We want this project to be a place where contributors can collaborate productively across technical backgrounds, languages, and experience levels.

Participants are expected to act with respect, clarity, and professionalism.

## Expected Behavior

- Be respectful in discussions, reviews, and issue threads
- Assume good intent, but stay precise and technical
- Focus criticism on ideas, code, and design decisions, not people
- Welcome contributors who are learning the codebase
- Share context when disagreeing so others can evaluate your reasoning

## Unacceptable Behavior

- Harassment, threats, or personal attacks
- Discriminatory or exclusionary language
- Repeatedly derailing technical discussions
- Publishing private information without permission
- Spam, trolling, or bad-faith participation

## Project-Specific Standards

- Keep claims about architecture, security, and parity grounded in evidence
- When raising a bug or design concern, include reproducible context when possible
- If you update English markdown docs, update the Chinese version too when one exists
- Respect the repository's focus on OpenClaw improvements derived from the published specification

## Enforcement

Project maintainers may edit, remove, reject, or moderate contributions, comments, issues, or pull requests that do not align with this code of conduct.

## Reporting

If you experience or observe behavior that violates this code of conduct, open a private communication channel with the maintainers if available, or file a minimal issue without personal details and request follow-up.

## Scope

This code of conduct applies to repository discussions, issues, pull requests, code reviews, and other project spaces.
