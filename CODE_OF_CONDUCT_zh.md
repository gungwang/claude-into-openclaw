# 行为准则

## 我们的承诺

我们希望这个项目成为一个能让不同技术背景、语言背景和经验水平的贡献者高效协作的地方。

所有参与者都应以尊重、清晰和专业的方式交流。

## 期望行为

- 在讨论、评审和 Issue 交流中保持尊重
- 默认他人出于善意，但表达要准确、技术化
- 批评应针对观点、代码和设计决策，而不是针对个人
- 欢迎正在学习代码库的新贡献者
- 当你不同意某个观点时，请提供上下文，便于他人评估你的理由

## 不可接受的行为

- 骚扰、威胁或人身攻击
- 歧视性或排斥性的语言
- 反复打断或偏离技术讨论
- 未经许可公开他人隐私信息
- 垃圾信息、恶意挑衅或缺乏诚意的参与

## 本项目的额外标准

- 关于架构、安全性和对等性的判断，应尽量基于证据
- 提出 Bug 或设计问题时，尽可能提供可复现的上下文
- 如果你更新了英文 Markdown 文档，在存在中文版时也应同步更新
- 尊重本仓库聚焦于基于公开规范推进 OpenClaw 改进这一目标

## 执行方式

项目维护者可以编辑、删除、拒绝或限制不符合本行为准则的贡献、评论、Issue 或 Pull Request。

## 举报方式

如果你遭遇或发现违反本行为准则的行为，请在有条件的情况下通过私下渠道联系维护者，或者提交一个不包含个人隐私信息的最小化 Issue 并请求后续处理。

## 适用范围

本行为准则适用于仓库讨论、Issue、Pull Request、代码评审以及其他项目相关空间。
